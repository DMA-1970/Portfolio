Development Team Project: Cloud Infrastructure & Framework Report

This assignment has a 20% weighting and a word count equivalent of 1,000 words. Submit by the end of Unit 6 (covers Weeks 1–6).

Coursework Brief

This team-based project assesses your understanding of foundational cloud computing concepts, cloud service models, deployment types, and cloud infrastructure design as introduced across Weeks 1–6. Each team (3–4 students) will collaborate on a report evaluating a cloud infrastructure framework, focusing on both theoretical understanding and applied design.

Coursework Tasks

  • Overview: Describe the chosen cloud framework, highlighting its core features and purpose.
  • Service Model Analysis: Explain how the framework supports different service models (IaaS, PaaS, SaaS).
  • Deployment Types: Discuss applicability across public, private, and hybrid cloud environments.
  • Infrastructure Design: Outline steps for designing a basic infrastructure using the framework (compute, storage, networking).
  • Case Study: Provide a real-world example of the framework in use and evaluate its impact on organisational efficiency and cost.

Evaluation Criteria

  • Conceptual Understanding (25%): Demonstrates clear understanding of cloud models and frameworks.
  • Analytical Skills (25%): Effectively analyses framework features, service models, and deployment types.
  • Technical Accuracy (25%): Accurately outlines infrastructure design with relevant details.
  • Collaboration & Cohesion (25%): Evidence of teamwork with a cohesive report structure.

Learning Outcomes

By completing this assignment, students will be able to:

  • Identify and explain the fundamental concepts behind cloud computing.
  • Identify and evaluate different cloud infrastructure solutions.
