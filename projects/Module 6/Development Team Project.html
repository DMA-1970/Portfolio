Development Team Project: Cloud Infrastructure & Framework Report 20% weighting

Submit by end of Unit 6 (covers Weeks 1–6) • Word count equivalent: 1,000 words

Coursework Brief

This team-based project assesses your understanding of foundational cloud computing concepts, cloud service models, deployment types, and cloud infrastructure design introduced across Weeks 1–6. Your team will collaborate on a report that evaluates a cloud infrastructure framework, demonstrating both theoretical understanding and applied design.

Team size: 3–4 students  •  Submission: One team report (1,000 words ±10%) with references in Harvard “Cite Them Right”.

Coursework Task

Choose one framework: AWS CloudFormation, Microsoft Azure Resource Manager (ARM), or Google Cloud Deployment Manager.

  1. Overview — Briefly describe the chosen framework, its core features and purpose.
  2. Service Model Analysis — Explain how it supports IaaS, PaaS, and (where relevant) SaaS models.
  3. Deployment Types — Discuss applicability across public, private, and hybrid environments.
  4. Infrastructure Design — Outline steps to design a basic infrastructure using the framework (compute, storage, networking, identity, and observability).
  5. Case Study / Real-World Example — Present a credible example of the framework in use; evaluate impact on efficiency, resilience, security, and cost.

Deliverables

Item Details Marks hint
Team Report (PDF/Docx) ~1,000 words, Harvard references, UK English. Clear headings matching the task. All criteria
Architecture Diagram High-level diagram of the designed infrastructure (label components and data flows). Technical Accuracy
Appendix (optional) Code snippets / templates (e.g., ARM/Bicep, CloudFormation, DM YAML), cost table. Technical Accuracy
Collaboration Evidence Brief roles table (who did what), meeting log, version control screenshots. Collaboration & Cohesion

Tip: Keep the main report concise; place long code or screenshots in an appendix.

Evaluation Criteria (Rubric)

Criterion Weight What good looks like
Conceptual Understanding 25% Accurate definitions of cloud models & frameworks; correct use of terminology; clear scope & assumptions.
Analytical Skills 25% Balanced evaluation of features, service models, deployment types; considers trade-offs and constraints.
Technical Accuracy 25% Sound design steps and components; secure, resilient, and observable architecture; plausible cost awareness.
Collaboration & Cohesion 25% Consistent voice and structure; clear division of labour; integrated sections; proper referencing.

  • Identify and compare different Cloud solutions and their infrastructure frameworks.

Team Roles Template

  • Lead Writer / Editor: integrates sections, ensures Harvard referencing.
  • Research Lead: gathers sources, case study evidence, cost notes.
  • Technical Designer: drafts architecture diagram and IaC snippets.
  • Case Study Lead: validates real-world example and impact analysis.

Suggested Timeline (Week 6)

  1. Day 1–2: Select framework, agree roles, outline sections.
  2. Day 3–4: Draft Overview, Service Models, Deployment Types.
  3. Day 5: Draft Infrastructure Design + diagram.
  4. Day 6: Case Study & impact evaluation; assemble appendix.
  5. Day 7: Edit, references, final QA, export PDF & submit.

Marking Tips

  • Signpost clearly with headings that mirror the task.
  • Show trade-offs (e.g., ARM vs. Bicep; CFN macros vs. modules; DM deprecation path).
  • Security & compliance: identity boundaries, encryption, logging.
  • Cost awareness: use conservative, referenced estimates if included.
